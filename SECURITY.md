# セキュリティガイド

このドキュメントでは、Claude Code Docker環境のセキュリティアーキテクチャと推奨事項を説明します。

## セキュリティアーキテクチャ

### ホスト環境との分離

このテンプレートはホスト環境への影響を最小限にするため、以下の設計を採用しています。

| リソース | 方式 | 説明 |
|----------|------|------|
| Claude認証データ | `claude-auth/`（ローカル） | ホストの `~/.claude` をマウントせず、プロジェクト専用の認証データを使用 |
| SSH鍵 | `ssh-keys/`（ローカル、RO） | ホストの `~/.ssh` をマウントせず、プロジェクト専用鍵を使用 |
| Git設定 | `config/.gitconfig`（ローカル、RO） | ホストの `~/.gitconfig` をマウントせず、プロジェクト専用設定を使用 |
| 設定ファイル | `config/`（RO） | 読み取り専用マウントでコンテナからの改ざんを防止 |
| MCPサーバー | `mcp-servers/`（RO） | 読み取り専用マウントでコンテナからの改ざんを防止 |

### コンテナセキュリティ

- **`no-new-privileges`**: コンテナ内でのプロセス権限昇格を防止
- **`cap_drop: ALL`**: 全てのLinux Capabilityを削除
- **`cap_add`**: 最低限必要な `CHOWN`, `SETUID`, `SETGID` のみ付与
- **SSH Agent非共有**: ホストのSSH Agentソケットをマウントしないため、ホスト上の全SSH鍵へのアクセスを遮断

## 残存リスク

### 1. ネットワークアクセス（高）

コンテナからのネットワークアクセスはデフォルトで制限されていません。`--dangerously-skip-permissions` で実行されるClaude Codeは、任意の外部通信が可能です。

**対策**: 必要に応じてDockerネットワーク制限を適用してください（下記参照）。

### 2. claude-auth/ 内の認証トークン（中）

`claude-auth/` にはClaude Codeの認証トークンが保存されます。このディレクトリへのアクセス権を持つ者は認証情報を利用できます。

**対策**:
- `claude-auth/` のファイルパーミッションを適切に設定（700）
- 不要になった場合は `claude-auth/` の内容を削除
- `.gitignore` に含まれておりリポジトリにはコミットされません

### 3. workspace/ への書き込み（中）

コンテナはワークスペースに対してRWアクセスを持ちます。開発に必要ですが、悪意のあるコード実行によりワークスペース内のファイルが改ざされるリスクがあります。

**対策**: 重要な変更は `git diff` で確認し、コミット前にレビューしてください。

### 4. sudo権限（低〜中）

コンテナ内の `claude` ユーザーはsudo権限を持ちます（パッケージインストール等に必要）。`cap_drop` と `no-new-privileges` により、ホストへの影響は制限されています。

## ネットワーク制限の参考設定

### 方法1: Docker内部ネットワークの使用

`docker-compose.override.yml` で外部アクセスを遮断できます:

```yaml
services:
  claude-dev:
    networks:
      - internal

networks:
  internal:
    internal: true  # 外部アクセスを遮断
```

> 注意: この設定ではClaude CodeのAPI通信も遮断されるため、別途プロキシの設定が必要です。

### 方法2: 特定ホストのみ許可

iptablesやDockerのネットワーク設定で、APIエンドポイント（`api.anthropic.com`）のみ許可する方法もあります。

## 推奨事項

1. **SSH鍵は専用鍵を使用**: ホストの鍵をコピーするのではなく、プロジェクト専用の鍵ペアを生成して使用してください
2. **config/.gitconfig を編集**: 名前とメールアドレスを設定してください
3. **不要な認証データは削除**: プロジェクト終了時に `claude-auth/` と `ssh-keys/` をクリーンアップしてください
4. **定期的にSSH鍵をローテーション**: プロジェクト専用鍵を定期的に更新してください
5. **.env に機密情報を入れすぎない**: 必要最低限の情報のみ記載してください
